<Section icon={Settings} title="Requisitos Essenciais para o Sistema ERP">

Para alcançar o sucesso na **automação da Zenith Inc**, o sistema ERP proposto deve atender a um conjunto rigoroso de requisitos funcionais. Esses requisitos são divididos em recursos "Indispensáveis" (Must-Have), essenciais para as operações diárias, e recursos "Desejáveis" (Nice-to-Have), que fornecem valor adicional e "personalidade" ao sistema. Uma das capacidades técnicas mais críticas é a habilidade do sistema de detectar e diferenciar automaticamente entre pedidos de compra (PO), notas de entrega de produtos e faturas.

Além disso, o sistema deve suportar uma força de trabalho global. Isso inclui uma interface robusta em vários idiomas para acomodar a equipe diversificada da empresa e seus parceiros internacionais. A automação do processo de registro de fornecedores, incluindo a geração de IDs exclusivos e o estabelecimento de uma hierarquia organizacional para autorizações, é um pilar fundamental desta transformação digital.

<Section icon={Target} title="O Processo de Fluxo de Trabalho Automatizado">

A jornada do usuário dentro do novo sistema automatizado foi projetada para ser tão intuitiva quanto as plataformas de varejo modernas, como o eBay. Desde a colocação inicial do pedido até a notificação final de entrega, cada etapa é rastreada e validada pelo ERP. Esse fluxo de trabalho garante que os dados do cliente sejam tratados com segurança e que todas as transações sejam autorizadas por meio de um processo de autenticação de várias etapas.

A automação começa na fase de aplicação, onde os clientes preenchem as quantidades e locais dos itens. Uma vez finalizado, o sistema aciona um processo de autorização de cartão de crédito seguido por uma autenticação de login segura para verificar a identidade do usuário.

<Steps>
  <Step num="1" title="Finalização do Pedido">
    Os clientes selecionam itens, quantidades e locais de entrega através da interface do ERP.
  </Step>
  <Step num="2" title="Autorização de Pagamento">
    O sistema solicita autorização do cartão de crédito para garantir que os fundos estejam disponíveis para a transação.
  </Step>
  <Step num="3" title="Autenticação do Usuário">
    O usuário fornece um nome de usuário e senha para autenticar o login para processamento seguro.
  </Step>
  <Step num="4" title="Processamento de Pagamento">
    Os clientes selecionam seu método de pagamento preferido e fornecem os detalhes finais de faturamento.
  </Step>
  <Step num="5" title="Notificação">
    Notificações automatizadas por e-mail ou telefone são enviadas ao cliente assim que o pacote entra na fase de entrega.
  </Step>
</Steps>

<Callout type="info" title="Experiência do Usuário">
O objetivo deste fluxo de trabalho é espelhar a simplicidade dos sites de e-commerce de alto nível, mantendo a segurança robusta de um ERP de nível empresarial.
</Callout>

</Section>
