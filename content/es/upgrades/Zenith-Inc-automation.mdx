<Section icon={Settings} title="Requisitos principales para el sistema ERP">

Para lograr una **automatización exitosa de Zenith Inc**, el sistema ERP propuesto debe cumplir con una serie de requisitos funcionales rigurosos. Estos requisitos se dividen en características "imprescindibles", que son esenciales para las operaciones del primer día, y características "deseables", que proporcionan valor adicional y un valor añadido al sistema. Una de las capacidades técnicas más críticas es la capacidad del sistema para detectar y diferenciar automáticamente entre órdenes de compra, notas de entrega de productos y facturas.

Además, el sistema debe admitir una fuerza laboral global. Esto incluye una interfaz multilingüe sólida para acomodar al personal diverso de la empresa y a sus socios internacionales. La automatización del proceso de registro de proveedores, incluida la generación de ID únicos y el establecimiento de una jerarquía organizacional para las autorizaciones, es la piedra angular de esta transformación digital.

<Callout type="warning" title="Requisito crítico">
El sistema debe detectar automáticamente los tipos de documentos (OC, nota de entrega, factura) para mejorar las capacidades de integración y respaldar la conciliación automatizada.
</Callout>

<Section icon={Target} title="El proceso de flujo de trabajo automatizado">

El recorrido del usuario dentro del nuevo sistema automatizado está diseñado para ser tan intuitivo como las plataformas minoristas modernas como eBay. Desde la colocación inicial del pedido hasta la notificación de entrega final, cada paso es rastreado y validado por el ERP. Este flujo de trabajo garantiza que los datos del cliente se manejen de forma segura y que todas las transacciones estén autorizadas a través de un proceso de autenticación de varios pasos.

La automatización comienza en la fase de solicitud, donde los clientes completan las cantidades y ubicaciones de los artículos. Una vez finalizado, el sistema activa un proceso de autorización de tarjeta de crédito seguido de una autenticación de inicio de sesión segura para verificar la identidad del usuario.

<Steps>
  <Step num="1" title="Finalización del pedido">
    Los clientes seleccionan artículos, cantidades y ubicaciones de entrega a través de la interfaz ERP.
  </Step>
  <Step num="2" title="Autorización de pago">
    El sistema solicita la autorización de la tarjeta de crédito para garantizar que los fondos estén disponibles para la transacción.
  </Step>
  <Step num="3" title="Autenticación de usuario">
    El usuario proporciona un nombre de usuario y una contraseña para autenticar el inicio de sesión para un procesamiento seguro.
  </Step>
  <Step num="4" title="Procesamiento de pago">
    Los clientes seleccionan su método de pago preferido y proporcionan los detalles de facturación finales.
  </Step>
  <Step num="5" title="Notificación">
    Se envían notificaciones automáticas por correo electrónico o teléfono al cliente una vez que el paquete está en la fase de entrega.
  </Step>
</Steps>

<Callout type="info" title="Experiencia de usuario">
El objetivo de este flujo de trabajo es reflejar la simplicidad de los sitios de comercio electrónico de primer nivel mientras se mantiene la seguridad robusta de un ERP de nivel empresarial.
</Callout>

</Section>
